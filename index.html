With Tidal we use text to generate music


              d1 
                $ cat [ 
                    n "[2 0 5] [4 [0@ 0]]",
                    n "[2 0 7] [5 [0@ 0]]"
                  ]
                # sound "supervibe"

Tidal is a notation or language for describing patterns

Patterns are what separates music from noise:
  • Rhythm
  • Melody / Harmony (pitch)
  • Dynamics (soft / loud)
  • Voicing/timbral patterns like call/response
  • Structural patterns like verse/chorus/bridge

If this is the first time you have heard the word 'string' in this context:

When we are talking about code, we're dealing with two kinds of text (at least...), so we need to distinguish between:

  • Source Code - any instructions that we write, eg functions and variables
  • Strings - text that is data, that the source code interacts with

If this doesn't make much sense now, just remember a "string is enclosed with double quotes" in Tidal 


              d1 
                $ cat [ 
                    n "[2 0 5] [4 [0@ 0]]",
                    n "[2 0 7] [5 [0@ 0]]"
                  ]

These strings are patterns in Tidal's notation
