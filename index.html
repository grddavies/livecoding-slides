What is Tidal?

Tidal is software for making music

What makes it different?

What makes it different?

Traditional western music is written using notes on a stave

Features:
  • Read left-to-right
  • The height of a note on the stave represents the pitch
  • The note's shape (ie with/without tail, head filled or hollow) represents duration, along with any dots after the note

What makes it different?

Ableton uses a 'piano-roll' to represent rhythms and melodies

Features:
  • Read left-to-right, but regularly we loop a clip
  • The height of a note on the stave represents the pitch
  • The note's length represents duration

What makes it different?

With Tidal we use text to generate music:


              d1 
                $ cat [ 
                    note "[2 0 5] [4 [0@ 0]]",
                    note "[2 0 7] [5 [0@ 0]]"
                  ]
                # sound "supervibe"

This system for describing music is called mini-notation

Using mini-notation we can build patterns

Features:
  • Patterns are read left-to-right
  • Pattern combination means that you need to read the entire expression to know how something will sound
  • Pitch can be represented with a number or a letter (more on this later)
  • Note length is determined by subdividing cycles according to any groupings in the notation

Patterns

Cycles, Patterns, Bars and Beats

Patterns

  • Typically we break music up into phrases, bars and beats
  • In Tidal, the cycle is the fundamental unit. We break cycles down into bars and beats using patterns
  • Using Tidal's mini-notation, we can build patterns and combine them into musical phrases
Patterns are what separates music from noise:
  • Rhythm
  • Melody / Harmony (pitch)
  • Dynamics (soft / loud)
  • Voicing/timbral patterns like call/response
  • Structural patterns like verse/chorus/bridge

Let's break down the pattern we saw earlier...


              d1 
                $ cat [ 
                    note "[2 0 5] [4 [0@ 0]]",
                    note "[2 0 7] [5 [0@ 0]]"
                  ]
                # sound "supervibe"
            

              d1

'd1' is a pattern name, you can think of it as an output channel

On a signle output channel you can send any number of sounds, control its volume (gain) and apply effects (apply to all sounds on the channel)

d1-16 are conventionally used, but we could have used a number or a string to name the pattern 


              p 42
                $ ...
              -- or
              p "patty"
                $ ...

If this is the first time you have heard the word 'string' in this context:

When we are talking about code, we're dealing with two kinds of text (at least...), so we need to distinguish between:

  • Source Code - any instructions that we write, eg functions and variables
  • Strings - text that is data, that the source code interacts with

If this doesn't make much sense now, just remember a "string is enclosed with double quotes" in Tidal 


              d1 
                $ cat [ 
                    note "[2 0 5] [4 [0@ 0]]",
                    note "[2 0 7] [5 [0@ 0]]"
                  ]

These strings are patterns in Tidal's notation

The 'note' function, converts numbers in the patterns into notes, +1 for each semitone

The square brackets in the pattern group the notes, and give the pattern its rhythmic structure. More on creating rhythms later... 


              d1 
                $ cat [ 
                    note "[2 0 5] [4 [0@ 0]]",
                    note "[2 0 7] [5 [0@ 0]]"
                  ]

'cat' is a function that acts on a list of patterns

It concatenates multiple patterns, playing them one after the other 


              d1 
                $ cat [ 
                    note "[2 0 5] [4 [0@ 0]]",
                    note "[2 0 7] [5 [0@ 0]]"
                  ]
                # sound "supervibe"

Finally, we add another pattern which describes which sound should play the melody described before it

This describes exactly the same pattern as before


              d1 
                $ sound "supervibe"
                + cat [ 
                    note "[2 0 5] [4 [0@ 0]]",
                    note "[2 0 7] [5 [0@ 0]]"
                  ]

There is not fixed order in which you need to define your patterns, but order does influence the output:

Notice how we combine the patterns with a `+` rather than a `#` this time
